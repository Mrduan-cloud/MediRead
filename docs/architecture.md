## 2. 报告解析 Agent

| 步骤            | 说明                                                                |
| --------------- | ------------------------------------------------------------------- |
| OCR             | PaddleOCR 中文模式，支持 image / pdf；PDF 走 `pdf2image` 分页       |
| 版式识别        | 通过医院 / 体检中心关键字判定模板（华西 / 美年 / 爱康 / 慈铭 / 平安 / 通用兜底） |
| 6 字段抽取      | `指标名 / 数值 / 单位 / 参考范围 / 性别 / 年龄`                     |
| 指标归一化      | 中英文别名 → 标准名（`app/data/synonyms.json`）                     |
| 强约束          | Pydantic + JSONSchema 双层校验；结构化 JSON 归档 MinIO              |

## 3. 医学解读 Agent

```
ReAct: Thought → Action(kb_retrieve / joint_analysis / risk_grade) → Observation → ...
```

- **多指标联合分析**：避免「单指标偏离一律打高风险」
  - `ALT + AST + GGT` → 肝损伤迹象
  - `TC + TG + LDL-C` → 血脂代谢
  - `HGB + RBC` → 贫血倾向
- **四级风险**：轻度偏离 / 关注观察 / 建议复查 / 建议就医
- **四级就医**：无需 / 全科 / 专科 / 急诊
- **危急值兜底**：肿瘤标志物显著升高等 → 强制「建议尽快就诊专科」

## 4. RAG 链路

```
权威医学资料 ──切分──▶ 元数据增强 (项目/系统/风险/适用人群)
                              │
                              ▼
                       Embedding (BGE) ──▶ Milvus
                              │
查询 ─┬─ BM25 ──┐             │
      │         │             │
      └─ BGE ───┘──RRF 融合──▶ Cross-Encoder 精排 ──▶ Interpreter
```

## 5. 私有化部署

- **Ollama** 跑 **GLM-4-9B-Chat 量化版**（q4_0），所有体检数据均在内网处理
- 用户上传的报告与解读结果归档至本地 MinIO，不出网
- API 网关层只做鉴权 / 限流 / 审计日志
